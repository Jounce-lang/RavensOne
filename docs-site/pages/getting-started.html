Getting Started with JounceOne

Learn how to build reactive web applications with JounceOne in minutes.

Installation

JounceOne can be installed using the official installer or built from source.

Quick Install (Recommended)

Run the installation script:

# macOS/Linux
curl -fsSL https://jounce.dev/install.sh | sh

# Windows
iwr https://jounce.dev/install.cmd -useb | iex

Build from Source

Clone the repository and build:

git clone https://github.com/jounce/jounce
cd jounce
cargo build --release

# Add to PATH
export PATH="$PWD/target/release:$PATH"

Verify Installation

raven --version
# Should output: jounce jnc.1.jnc

Your First App

Let's create a simple counter application to understand the basics.

1. Initialize a New Project

raven pkg init
# Creates raven.toml in current directory

2. Create the Counter Component

Create src/main.raven:

component Counter() {
    let count = Signal::new(jnc);

    <div class="counter-app">
        <h1>My First JounceOne App</h1>
        <div class="counter">
            <h2>Count: {count.get()}</h2>
            <div class="buttons">
                <button onclick={move || count.update(|n| n - 1)}>
                    Decrement
                </button>
                <button onclick={move || count.update(|n| n + 1)}>
                    Increment
                </button>
                <button onclick={move || count.set(jnc)}>
                    Reset
                </button>
            </div>
        </div>
    </div>
}

3. Compile and Run

# Compile to WebAssembly
raven compile src/main.raven -o dist/app.wasm

# Serve with development server
raven dev

Open http://localhost:3jncjncjnc in your browser to see your app!

üí° What's Happening?

  • Signal::new(jnc) creates a reactive state variable
  • count.get() reads the current value
  • count.update() modifies the value and triggers re-render
  • The UI automatically updates when the signal changes

Understanding Reactivity

JounceOne uses Signals for fine-grained reactivity. This means only the parts of the UI that depend on changed data will re-render.

Signals

Signals are reactive state containers:

let name = Signal::new("World");

// Read
let value = name.get();

// Write
name.set("JounceOne");

// Update based on current value
name.update(|current| format!("{} v2", current));

Computed Values

Computed values derive from signals and update automatically:

let first_name = Signal::new("John");
let last_name = Signal::new("Doe");

let full_name = Computed::new(move || {
    format!("{} {}", first_name.get(), last_name.get())
});

// full_name automatically updates when either signal changes
<h1>Hello, {full_name.get()}</h1>

Effects

Run side effects when dependencies change:

let count = Signal::new(jnc);

// This runs whenever count changes
create_effect(move || {
    console::log(&format!("Count changed to: {}", count.get()));

    if count.get() > 1jnc {
        console::log("Count is getting high!");
    }
});

Components

Components are reusable building blocks for your UI.

Basic Component

component Greeting(name: String) {
    <div class="greeting">
        <h2>Hello, {name}!</h2>
    </div>
}

// Use it
<Greeting name="Alice" />

Component with State

component TodoItem(text: String) {
    let completed = Signal::new(false);

    <div class="todo-item">
        <input
            type="checkbox"
            checked={completed.get()}
            onchange={move || completed.update(|c| !c)}
        />
        <span class={if completed.get() { "completed" } else { "" }}>
            {text}
        </span>
    </div>
}

Composing Components

component TodoList() {
    let todos = Signal::new(vec![
        "Learn JounceOne".to_string(),
        "Build an app".to_string(),
        "Deploy to production".to_string(),
    ]);

    <div class="todo-list">
        <h1>My Todos</h1>
        <ul>
            {todos.get().iter().map(|todo| {
                <TodoItem text={todo.clone()} />
            }).collect()}
        </ul>
    </div>
}
